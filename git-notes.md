# Git
Git — это система контроля версий для отслеживания изменений в проектах

Инициализация и клонирование:
git init: Инициализирует новый Git-репозиторий.
git clone <url>: Клонирует существующий репозиторий удаленно. 

Основные команды для работы с файлами:
git status: Показывает состояние файлов (изменены, добавлены, не отслеживаются).
git add <file> / git add .: Добавляет файлы в индекс (staging area).
git commit -m "Сообщение": Сохраняет изменения из индекса в историю с сообщением.
git diff: Показывает различия между рабочим каталогом, индексом и последним коммитом.
git reset <file>: Отменяет git add, удаляет файл из индекса.
git rm <file>: Удаляет файл из рабочего каталога и индекса.
git mv <old> <new>: Перемещает/переименовывает файл (эквивалентно git rm и git add). 

Работа с историей и ветками:
git log: Показывает историю коммитов.
git branch: Список веток, создание новой ветки.
git checkout <branch> / git switch <branch>: Переключается на указанную ветку.
git merge <branch>: Сливает указанную ветку в текущую.
git rebase <branch>: Перебазирует текущую ветку на другую (переписывает историю).
git stash: Временно сохраняет изменения, чтобы очистить рабочую директорию. 

Работа с удаленными репозиториями:
git remote add origin <url>: Добавляет удаленный репозиторий (обычно origin).
git push origin <branch>: Отправляет коммиты в удаленный репозиторий.
git pull origin <branch>: Получает изменения из удаленного репозитория и сливает их.
git fetch: Получает изменения с удаленного репозитория, но не сливает их. 

Откат изменений:
git revert <commit>: Создает новый коммит, отменяющий изменения указанного коммита.
git reset --hard HEAD: Отменяет все локальные изменения и коммиты (осторожно, данные теряются). # -Git
